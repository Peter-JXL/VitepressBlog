# 600.RSS 阅读器推荐

本篇文章将简单介绍一些好用的、流行的 RSS 阅读器，希望你能看到喜欢的 😘。

‍‍<!-- more -->

## RSS 阅读器的分类

目前市面上有很多 RSS 的阅读器，但都可以简单地分为：

* 在线阅读器：可以多平台使用和同步的阅读器
* 本地阅读器：就是在本地阅读 RSS 的软件

至于选择哪一款，则见仁见智了：

* 在线阅读器能同步，但是一般在会分为免费版（一般会限制订阅数量、刷新时间）和收费版
* 对于我来说，我平时都是围绕着电脑工作的，选择本地阅读器问题也不大

当然，也可以结合使用：使用本地 RSS 客户端来浏览在线 RSS 帐户，这样就可以在任意设备都阅读 RSS，并且能互相同步，这可能是最佳方案了。
‍
使用在线阅读器还有一个好处：对于本地阅读器来说，只有一直开着电脑或手机才能获取到更新，如果有些平台更新频率高，一天更新了 100 篇文章，而 RSS 的输出数量是有限的，如果一天后再打开本地阅读器，刷新后只能看到最新的几篇了（当然这种情况比较少见）。

而用在线阅读器没这个问题，可以设置定时刷新。

## 本地阅读器
‍
#### Fluent Reader

![](https://image.peterjxl.com/blog/image-20240809113425-pdkkjao.png)​

首推的就是 Fluent Reader 了，它是一款跨平台的开源 RSS 阅读器，具有简洁直观的界面和丰富的功能。

官网：[https://hyliu.me/fluent-reader](https://hyliu.me/fluent-reader)

开源在：[https://github.com/yang991178/fluent-reader](https://github.com/yang991178/fluent-reader)

下载地址：

* Microsoft 或 Apple 的应用商店（在官网有下载地址，或者去应用商店内搜）
* [GitHub Releases](https://github.com/yang991178/fluent-reader/releases)


特点：

1. **多平台支持**：可以在 Windows、MacOS 和 Linux 等主流操作系统上运行，也提供了 Android 和 iOS
2. **丰富的功能**：除了基本的 RSS 阅读功能外，还支持多个标签、分类、快捷键操作、搜索、离线阅读等功能。
3. **自定义界面**：用户可以根据自己的喜好调整界面，包括主题颜色、字体大小、阅读布局等。
4. **同步功能**：支持和在线 RSS 应用同步，如后续会介绍的 Feedly、Inoreader 和 The Old Reader 等。
‍
我找过很多款 RSS 本地阅读器，Fluent Reader 可以说是最漂亮最好用的了，强烈推荐使用，一步到位。

## 在线阅读器

### Feedly

![](https://image.peterjxl.com/blog/image-20240809174915-nbe8hrv.png)​

官网：[https://feedly.com](https://feedly.com) （需魔法上网）

Feedly 是一款备受欢迎的、老牌的（2008 年推出）在线 RSS 阅读器，特点：

* 由于是网页端应用，因此可以跨设备同步
* 界面优雅，字体和图片都做了优化，很美观
* 该有的功能都有：订阅、搜索，将信息保存到稍后阅读列表
* 大部分功能是免费的，一些高级功能收费，例如和第三方（印象笔记、WordPress）集成

### Inoreader

![](https://image.peterjxl.com/blog/image-20240809175134-0k5abqa.png)​

官网：[inoreader.com](https://www.inoreader.com)（需魔法上网）

国内版：[innoreader.com](https://www.innoreader.com)（多了个 n）

Inoreader 是另一个非常受欢迎的在线 RSS 阅读器，特点：

* 由于是网页端应用，因此可以跨设备同步
* 可以用标签、文件夹、规则等整理 RSS 源
* 支持离线阅读
* 免费版有部分广告，并且有订阅源数量限制（150 条，一般也够用了）

## 自建 RSS 在线阅读器

使用第三方的 RSS 在线阅读器，多多少少有几个缺点：

* 和平台绑定较深，如果平台倒闭/被墙了，那么就访问不到了；如果没有及时备份 RSS 源，那么数据就丢失了
* 不少功能都是收费的，并且有订阅数量限制，而自建的话没有这个问题.
* 自建不仅可以使数据更可控，还可以用丰富的插件可以满足各种各样的需求，比如全文内容提取、Fever API 模拟、DOM 操控、繁体转简体......
‍
当然，自建的话也有缺点：

* 涉及到一些服务器相关知识，有门槛
* 如果使用传统服务器，那么成本也不低
* 得考虑网络安全的问题，跨平台的问题（一般需要绑定一个域名）
‍

![](https://image.peterjxl.com/blog/image-20240809211141-kjmc5xt.png)​

官网：[tt-rss.org](https://tt-rss.org)

开源在 GitHub：[https://github.com/torne/Tiny-Tiny-RSS](https://github.com/torne/Tiny-Tiny-RSS)

功能：

该项目基于 PHP，是很早之前就开发的项目，我看很久没更新过了，不知道还有没在维护，官网文档较少。




### fusion

![](https://image.peterjxl.com/blog/image-20240503111005-dvoc336.png)​

开源在 GitHub：[https://github.com/0x2E/fusion](https://github.com/0x2E/fusion)

一个 Go + Svelte 开发的 RSS 聚合和阅读器。轻量、简洁、自建友好。

### INK RSS

![](https://image.peterjxl.com/blog/image-20240811103915-u8posa2.png)​

基于 Serverless 的 RSS 通知服务，开源在 GitHub：[https://github.com/pureink/inkrss](https://github.com/pureink/inkrss)

特点：

* 部署在 CloudFlare Workers，无需管理服务器和数据库
* 最短一分钟进行一次监测，及时性好
* 支持多种通知方式，包括但不限于 Telegram、Bark、微信
* 支持网页和 Telegram 两种订阅方式

该项目紧紧抓只了“通知”这个核心要义。

推荐阅读：[介绍一款可免费搭建属于自己的 RSS 服务 | 小嘉的部落格](https://blog.imzjw.cn/posts/inkrss/#%E6%80%BB%E7%BB%93)
