# 「以图搜图」最全教程，帮你找到 90% 的图片出处

​![](https://image.peterjxl.com/blog/image-20250211102829-a1gf119.png)​

之前说过会写一篇图片搜索的专题文章，现在它来了 😉

本文会介绍不少关于图片搜索的一些网站、技巧、工具等，还有一些我个人的经验，相信能帮到你。

## 开始之前

搜图，其实可以分为两个场景：以图搜图，以及找素材图片。

本文的重点在于以图搜图，就是搜索相似图片或完全相同的图片，常用来寻找现有图片的原始出处，或者低分辨率缩略图的原始大图。

至于找素材图片，我在讲搜索引擎技巧（足足讲了三篇文章）的时候已经讲过了，感兴趣的可以在页面底部的合集找。

此外，在本文中，部分场景需要魔法上网，请自行搜索解决。

好了，废话不多说，开始吧！
‍
## 简介

以图搜图，专业点叫反向图片搜索引擎。平常我们搜索图片是先输入关键词，然后找到符合关键词的图片；而反向图片搜索就是用图片去寻找符合图片的内容，比如类似的图片和该图片的信息。

这种场景还是不少见的：

* 找图：想找到关于这张图的更多信息，例如出处，无水印版，高清版
* 识图：识别图片上的信息，例如人物或物品。比如识别明星，花花草草，爆款商品等
* 来源：想知道这图出自哪部电影、动漫或漫画等
* ......
‍
## 先从本地搜

我经常遇到这样的情况：找到一张图片后，发现它其实早就已经在自己的图片库里了，可以说是白找了。

所以，在找一张图片之前，先看看本地有没这个图片，是一个好习惯。

那么，怎么搜本地图片？首先，根据自己的印象去找是很低效的，推荐几个工具：

* [类视搜图](https://ls-ai.cn/localsearch/)：支持以图搜图，语义描述搜图，以色彩搜图，人脸搜图等，提供专业版
* [ImageSearch](https://github.com/ldqk/ImageSearch)：免费开源，操作简单，还可通过索引达到秒级得到结果，支持千万级数据
* [LocalMediaSearch](https://github.com/kaisen-wang/local-media-search)：免费开源，跨平台，支持通过文字和图片搜索本地图片和视频文件，支持 GPU 加速
* [Zookr](https://gitlab.com/kneels/Zookr)：老早的搜图工具了，但搜索速度较慢
* [嗅图狗](https://github.com/naturalcaduceus/picSniffer)：免费开源，支持多种搜索模式。但没索引机制，搜索起来较慢
* [链图云](https://www.lianty.com)：识图过程会产生大量的缩略图，占用空间。搜索速度较慢
* ......

类似的软件还有很多，还有网友自己深入研究原理，搭建 Docker 镜像，写了一篇十万字的教程：[向量数据库入坑：使用 Docker 和 Milvus 快速构建本地轻量图片搜索引擎 - 苏洋博客](https://soulteary.com/2022/09/24/use-docker-and-milvus-to-quickly-build-a-local-lightweight-image-search-engine.html)

个人看法：一般来说网上找些搜图软件就够用了，如果你要求比较高，再去研究也不迟。

## 用搜索引擎

谷歌搜图：https://www.google.com/imghp

综合各方面来讲，谷歌识图无疑是最强的，数据库丰富，识别准确。

操作方法如下，点击相机图标：

​![](https://image.peterjxl.com/blog/image-20250209154750-eeh7c2l.png)​
‍

然后上传图片（或拖动照片、还可复制在线图片网址）即可识别：

​![](https://image.peterjxl.com/blog/image-20250209154745-sr32tw2.png)​
‍

一些小技巧：

* 如果一张图非常多人用，搜索结果会有很多页，得自个多翻几页，可能才找到出处。
* 把 UI 语言设置为所搜索关键词的语言，可以更好的匹配结果（搜索日语内容就设置为日语 UI，其他同理）
‍
还有国人搭建了谷歌识图镜像站，无需魔法上网即可使用，谷粉识图：https://gfsoso.99lb.net/.html

​![](https://image.peterjxl.com/blog/image-20250210123735-kdcq5xj.png)​
‍

但不得不说的是，谷歌识图最近越来越垃圾了......

Yandex 的效果也很不错，有时候比谷歌还好用，能搜到相似图片，也是很多人强推的一个，不用魔法上网，网页布局比谷歌好

官网：https://yandex.com/images/

不过该英文版的网站最近好像不能用了，可以换成俄区的：https://yandex.ru/images/

​![](https://image.peterjxl.com/blog/image-20250209173247-qim2w5y.png)​
‍

还有很多其他搜索引擎也提供搜图服务，建议搭配使用：

* [必应可视化搜索](https://www.bing.com/visualsearch)：微软出品，支持植物、商品、家具、狗狗、人物、建筑、地标等特色搜索。建议用国际版
* [百度搜图](https://image.baidu.com/)：一般，搜国内图片的话还行
* [搜狗识图](http://pic.sogou.com/)：识图效果也可以，比较人性化，还做了“通用识图，人物识别，找高清大图” 等分类
* [360 识图](http://image.so.com/)：试过几次，不太好用，只能说可以试试，不要抱太多希望

一般来说，首推谷歌和 Yandex。

但每个搜索引擎的算法不同，数据库不同，效果也不同，例如国内的图片用百度就还行。

如果一个图片搜不到，不妨换一个试试。

## TinEye

​![](https://image.peterjxl.com/blog/image-20250210104307-nzoh029.png)​

官网：https://www.tineye.com，需魔法上网。

老牌搜图网站，资源非常丰富，也是非常推荐的一个搜索引擎。提供浏览器扩展，可通过右键菜单直接使用。

缺点：无法搜索到分辨率差距悬殊的图片（即小图找大图效果不好），如果图片被整的面目全非（例如加水印）也很难搜到​​

## 搜二次元

以上介绍的是通用的图片搜索，还有一些针对特定场景的搜索，例如二次元。

有时候我们无意中看到了一张动漫截图或者好看的图片，很感兴趣，就想找到这部动漫或者高清原图出处来源。

搜索引擎毕竟是通用的图片搜索，搜动漫不妨用专业的网站：
‍
### Trace

网址：https://trace.moe，需魔法上网

可以说是最专业的动漫截图搜索引擎，找动漫必备！

支持上传截图（点击文件夹图标）、将图片拖入页面、粘贴图片链接三种方式进行搜索：

​![](https://image.peterjxl.com/blog/image-20250209161915-kelmgxc.png)​
‍

通过截图追溯获取对应的动画场景，并显示出该截图属于哪部动画作品，以及准确的集数，甚至精确到秒，并提供几秒的预览：

​![](https://image.peterjxl.com/blog/image-20250210165543-ivoic7j.png)​
‍

还提供浏览器扩展（可以在官网底部找到） Search Anime by Screenshot，直接右键搜图：

​![](https://image.peterjxl.com/blog/image-20250210122432-84h8s4s.png)​
‍

类似谷歌，有国人搭建了一个镜像站，调用 Trace 的服务，无需魔法上网：https://tools.miku.ac/what\_anime\_is\_this

​![](https://image.peterjxl.com/blog/image-20250210165806-sztfwd5.png)​
‍

提到这个，就不得不提其背后的原理和故事了。

该网站是香港开发者 soruly 的个人开源项目，作者收录了 1990 年之后的的正式发行的（日本）动画，然后经过一台双核 i7 处理器的机器分析之后，一共有几亿帧画面。

​![](https://image.peterjxl.com/blog/image-20250211153255-6ai3t6n.png)​
‍

搜索原理：根据你上传的图片到它的数据库里进行对比，从而得知出处。

更难能可贵的是，作者一直在维护更新，基本上常见的动画都可以找得到，很多新出的也能找到。
‍
注意，该网站对截图尺寸和质量要求比较严格，不太支持以下类型截图识别：

1. 1990 年之前的动画
2. 非日本动画，非正式发行的动画，即同人插图等
3. 有额外添加的黑边，裁切过的不完整截图，经过滤镜处理的
4. 画面过暗的，左右翻转的，分辨率过低的（须大于 320x180）
5. 加了文字的表情包
6. ....

网站的常见问题 FAQ 版块（https://trace.moe/faq）中写了很多内容，也可以在里面看看具体原因。
‍
对于次数也有一定的限制。用的人很多，服务器承受不住，作者曾在推特上发表声明：每个 IP 仅每小时限用 3 次。

​![](https://image.peterjxl.com/blog/image-20250211152636-4ua8mv2.png)​
‍

### SauceNAO

官网：https://saucenao.com，需魔法上网

非常知名的网站，对于动画、漫画、插画作品、二次元图片等有着非常不错的搜索效果，包含了如 Pixiv、Yande、Anime、Movies、MangaDex、ArtStation 等数十亿张图像的数据库。

我经常用它来搜 Pixiv 的图，Twitter 的图能搜一部分。

支持上传图片，或者复制图片的网络链接，还可以直接拖拽图片到网页上：

​![](https://image.peterjxl.com/blog/image-20250209172618-0agv32o.png)​
‍

上传图片，即可获得该图片的来源以及链接，甚至有 P 站画师 ID：

​![](https://image.peterjxl.com/blog/image-20250209172754-3sc9896.png)​
‍

该网站也提供支持浏览器扩展，可以在右键菜单里添加多个图片搜索引擎（需魔法上网），可在官网获取：https://saucenao.com/tools/

​![](https://image.peterjxl.com/blog/image-20250209172957-5if8978.png)​
‍

值得一提的是，在搜索结果页，也可以很方便地跳转到其他平台（例如谷歌，trace.moe）进行搜索，点击对应图标即可：

​![](https://image.peterjxl.com/blog/image-20250210102734-rb1q5ru.png)​
‍

该网站有部分广告，我这里已经屏蔽了。

## 搜其他场景

除了二次元，还有很多学术领域的搜图，例如：

* 搜商品：如果你看到某样东西，想买，但是不知道关键字，就用电商平台的搜索功能
* 搜矿物：中国地质调查局发布了「地质云」—— 智能识别矿物系统，结合了 AI
* 搜鸟类：http://www.nsii.org.cn/2017/birdApp.php
* 搜素材：[EveryPixel](https://www.everypixel.com)，素材反向搜索，集合了 50 个图库图，对设计师很有用
* [Berify](http://berify.com)：帮你找有谁盗用了你的图像或视频，例如律师用它来找诉讼证据等。收费网站
* [identify](http://imageidentify.com)：帮你辨别图像中的物体，并生成一系列描述图片内容的关键词和标签。结合了谷歌和亚马逊的识别系统，用了大量数据训练，可以达到 “人类的初级认知水平”
* 搜植物：

  * 专门识花的 App：形色
  * 问植物的 App：格物
  * https://ppbc.iplant.cn，由中国科学院植物研究所建设的植物图像库，收集整理了各种植物照片、图像资料，堪称最大的植物分类图片库，非常专业。可以通过关键词和图片进行搜索，准确率非常高。
* ......

如果你不知道某些领域该用什么网站搜，不妨直接用搜索引擎搜关键字，例如 “搜鸟网”、“识别植物的网站”...

## 搜图方法论

网上的搜图工具很多，但重要的是搜图的思路，以及多实践。

我自己总结了一些要点：

1. 根据场景进行搜索
2. 图片预处理后再搜
3. 补充图片信息
4. 终极大法：发动群众的力量
‍
### 根据场景搜

* 首先搞清楚想搜的图，是哪个方面的，二次元？商品？建筑？植物？然后选择对应的搜索引擎或网站。
* 如果是二次元，优先使用 SauceNAO，搜不到再跳转到其他网站即可
* 在动漫方面，trace.moe 更专业，很多刚上映的动漫都能搜出来
* 在推特方面，ascii2d 更专业，能搜到的图片更多
* ......
‍
值得一提的是，很多时候搜一张图，只会定位到该博主的主页，而不会直接给出具体的出处。

此时，就得自个往下翻了，一个个动态看下去（相信经常找图的人，会有这样的感触）。

例如，阿虚同学在教程 [以图搜图就仅仅是把图片甩给识图引擎？](https://mp.weixin.qq.com/s/bHQnw6zMPL3NeZNgJCY22A) 里就提过，找了去年的动态才找到：

​![](https://image.peterjxl.com/blog/image-20250210171116-ij8z3tz.png)​
‍

这里我提供一个思路：除了一条条往下翻，还可以下载该博主全部的图片，然后再用本地搜索。

我之前就写过类似的教程：[一个批量爬取微博数据的神器](https://www.peterjxl.com/weibo/crawler-intro/)，分享了一个 Python 的爬虫工具
‍
### 预处理

一般来说，以图找图，使用的图越完整越好。

当图处于很模糊 / 大量水印等状态时，搜索效果就不好了。

所以，遇到搜不到的情况，不妨试试预处理后，再搜。
‍
如果图太模糊或者太小，可以考虑对其进行清晰化，例如：

* Bigjpg：http://bigjpg.com，还提供 Windows、Mac、Android 下载，甚至还有微信小程序
* jpgHD：https://jpghd.com，人工智能老照片无损修复，提供 Windows、Mac，iOS 和 Android
* waifu2x：https://waifu2x.booru.pics
* PhotoShop：通道去噪点、模糊、高反差保留、图层叠加、锐化等等
‍
如果图片有水印，可能会误导搜索引擎，可以考虑用微软画图裁剪掉水印部分，或者用 Photoshop、AI 去水印。
‍
如果图不是完整的原图（被裁剪过，PS 等），尽量找到完整的图后，再进行二次搜索。

例如，想搜某人头像的动漫出处这种情况，直接用 trace.moe 或 IQDB 可能搜不到，因为只是部分截图。

即使用搜索引擎搜，一般出的结果也是头像：

​![](https://image.peterjxl.com/blog/image-20250210155548-z6o9tea.png)​
‍

所以，先想办法找个原图，例如用其他搜索引擎试试，找个尺寸更多、更完整的图：

​![](https://image.peterjxl.com/blog/image-20250210160101-msxa0it.png)​
‍

然后再试试用 SauceNAO 等专业网站：

​![](https://image.peterjxl.com/blog/image-20250210160131-y5l3eg7.png)​
‍

如果是动态图，部分搜索引擎可能不支持，得自个对动图取帧，例如用 ScreenToGIF，或者直接截图。

当然，取帧肯定也得选择画面清晰，和那种容易被人截图发布到网上的画面。
‍

## 使用第三方工具

搜图的一般过程为：

1. 打开搜图网站，上传图片，点击搜索；
2. 如果某个网站找不到，那就换一个网站，再次上传，再次点击搜索

可以看到，过程还是有点繁琐的，使用一些专门的搜图扩展和油猴脚本，有助于提高效率。推荐：

* [二箱](https://chromewebstore.google.com/detail/kidibbfcblfbbafhnlanccjjdehoahep)：包含了百度、谷歌、必应、Yandex、TinEye、IQDB、SauceNao 二次元画像七个搜索引擎
* [Search by Image](https://chromewebstore.google.com/detail/search-by-image/cnojnbdhbhnkbcieeekonklommdnndci)：精选扩展，30w 用户，还提供 Firefox、Bing、Opera 版本的扩展，以及 iOS ，开源
* [Search By Image](https://greasyfork.org/zh-CN/scripts/2998-search-by-image)：脚本，20w 安装量，作者一直在维护
‍
如果你用 Telegram，也可以用 Bot：

* Tracemoe Bot：https://t.me/WhatAnimeBot
* SauceNAO bot：https://t.me/SauceNAObot
* IQDB bot：https://t.me/iqdb_org_bot
* ASCII2D bot：https://t.me/ascii2d_bot
