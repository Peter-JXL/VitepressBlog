## 前言

大致原理：

1. 安装代理软件（以 Quantumult X 为例），并导入网友整理的规则
2. 后续所有流量都由代理软件接管，它会管理所有网络请求
3. 如果根据规则，发现某个网络请求是广告请求，则过滤该请求，从而导致广告加载失效

‍

主要步骤：

1. 获取一个美区的 Apple ID，然后在手机的 App Store 上登录该 Apple ID
2. 购买礼品卡（最低 10$，约 73.5￥）
3. 兑换礼品卡，购买并下载 Quantumult X（因为它是付费软件，7.99$ ）
4. 配置 Quantumult X，主要是生成证书 + 导入规则

‍

叠甲：

1. 会魔法上网最好
2. 本文涉及的互联网基础知识面比较广，且主要以实操为主，限于篇幅不会讲的太细，主要提供一个大致的方向。就好比我只是在教开车，而不讲解车的各个组件、为什么能动起来，那太复杂了。
3. 如果遇到一些细节的问题，建议自行搜索解决，我也会列一些详细的教程。
4. 别看过程只有四步，但每个步骤都挺麻烦的，估计能淘汰掉八成的网友。
5. 需要花费一点金钱和时间，但会收获很多。

‍

个人建议：不要想着一天内搞定，而是每天抽出点时间来弄。例如上述步骤分 4 天，甚至 4 周去做都行。

我自己就是每次受不了开屏广告时，就来弄一下，最后花了好久才成功的 🙃。

## 关于 Quantumult X

![](https://image.peterjxl.com/blog/image-20250127170016-pqi1zo3.png)

Quantumult X，俗称 “圈叉”，是一款功能强大的网络代理工具（支持 iOS，macOS），常被称为神器，主要为苹果系用户提供代理服务和网络优化，支持多种代理协议（Shadowsocks、VMess、Trojan 等），同时具备自定义策略、规则、MitM、脚本引擎等强大功能。

它是一个付费软件，不含税价格为 7.99 $。

由于众所周知的原因，它已从国区 App Store 下架，需要从其它国家地区的 App Store 购买。

如果你想好好地了解它，可以阅读：[Quantumult X Wiki Book](https://qx.atlucky.me/)，一本网友整理的小册子，非常详细，每个页面，每个按钮，各种强大功能都做了详细的图解。但我个人是不建议去看的，太多太复杂了，需要用到时再去看也不迟。

‍

关于读音，Quantumult X 的读音可以分为几部分：

- Quantum：一个英文单词，意思为「量子」，发音为 /ˈ kwɒntəm/
- Ult：发音为 "ult"（/ʌlt/），类似于单词 "ultimate"（终极）中的 “ult” 部分。
- X：发音为 "eks"（/ɛks/），和英文字母 X 的标准发音一样。

所以，Quantumult X 的发音大致是 "Quantum-ult eks" /ˈ kwɒntəmʌlt/

### 规则库的功能

先说说规则库。规则库就是判断某个网络请求的类型，并做相应处理的。

好的规则库可以：

- 去广告：包括各种 APP 内部广告、开屏广告、小程序广告，微信文章内广告
- 智能分流：网络请求自动匹配规则策略
- VIP 会员解锁：包括某音乐 VIP、某听书 VIP 等等
- 应用增强：例如 B 站自动换区，度盘开启倍速、Spotify 歌词翻译等等
- 网页优化：例如百度搜索去广告、谷歌搜索自动翻页等
- ......

注意：并不是所有 app 都有写对应的规则的，没有的需要自己去研究，甚至可以让 AI 帮你写一个规则库。

‍

### 规则库的重要性

杀毒软件中，杀毒效果依赖于核心病毒库的质量，去广告效果也依赖于其规则库的质量。

维护去广告规则库，其难度相比于病毒库难度高很多：全球几千万站点，每天上百亿次广告展现，频繁变更的广告植入方式，规则库维护难度是十分巨大的。

另外，互联网的核心思路是羊毛出在猪身上。站点运营需要成本，不在用户身上收钱，站点只能依靠其他方向获取收入，广告就是其中的主要来源。

因此，对于去广告技术，几乎所有站点都是抵制的（例如 KoolShare 论坛就是禁止类似技术讨论的）。为了保证收益，它们会根据去广告技术的发展不断变更广告插入方式，从而规避去广告软件的运作。

又比如，我经常关注的阮一峰的博客，就会做类似的限制。

![](https://image.peterjxl.com/blog/image-20250129100153-jscpzax.png)

所以，**广告与去广告一直是在不断斗争中发展的**。因为广告植入方式发生变化，一个有效的屏蔽规则很可能在一夜之间完全失效，或者某些站点可以让你即使屏蔽了广告展示，仍然要等待固定时间才能获取内容（例如某些视频站点）。

解决办法：

1. 尽量少更新 APP

2. 如果网友有更新规则，就及时可以导入

3. 理论上可以自己写规则（用 AI 辅助）

‍

从长远来说，广告有利于现在互联网站点的生存，对用户是有利的（如果所有站点都赚不到钱，最终只能关门，最终用户的利益也会受损）。

对于笔者自己，我认为需要屏蔽的实际上是恶意的，带有强迫，诱导性质的广告，木马钓鱼链接，隐私收集类的垃圾信息。对那些对日常生活，娱乐，学习帮助很大的站点，能出资支持，或者主动点击广告表达支持，帮助站点发展更好，才是双赢的做法。

‍

### 导入规则

首先，打开 APP，点击右下角的风车图标进入设置页面：

![](https://image.peterjxl.com/blog/image-20250605162412-ip8izrc.png)

‍

往下划到最后，会看到一个关于配置文件的地方。有多种导入配置的方法：

- 下载配置：从网址中下载配置文件
- 导入配置：从本机中导入配置文件（可以先从电脑上下载，然后发到手机上）

‍

![](https://image.peterjxl.com/blog/image-20250605162847-dodk4z5.png)

‍

我这里推荐用「导入配置」的方式，先下载下来，然后发送到手机上，然后点击导入配置，选择文件进行导入即可。

原因如下：

1. 首先，很多配置文件都是在 GitHub 上的，需要翻墙才能下载。
2. 但如果翻墙的话，日常使用过程中会感觉比较慢，因为翻墙后访问国内网络会有点慢。
3. 我自己平时是用一些不用翻墙，也能起部分作用的规则库，所以就不翻墙了。

具体的我们后面再说。

### 配置证书

由于我们要让 Quantumult X 接管流量，这是一件需要很高权限的事情。

因此我们得用「证书」，让 iOS 设备知道并授权给 Quantumult X。

进入设置页面，找到 MitM 的设置。

> MitM，全称 Man-in-the-Middle，俗称「中间人攻击」，它通过在设备上部署根证书解密和分析、处理 HTTPS 请求

通过 MitM，Quantumult X 才可以拦截、查看和修改 HTTPS 请求和响应，因此在使用中，它必须开启。

先点「生成证书」：

![](https://image.peterjxl.com/blog/image-20250129144844-amclxij.png)

‍

再点「配置证书」，会弹窗，我们点击确认：

![](https://image.peterjxl.com/blog/image-20250129145808-wl7ff1m.png)

‍

然后会跳转到 Safari 浏览器，并弹窗确认，我们点击允许：

![](https://image.peterjxl.com/blog/image-20250129145846-kt0shjm.png)

‍

然后会提示已下载：

![](https://image.peterjxl.com/blog/image-20250129145943-br146s5.png)

‍

我们返回到 iPhone 主页面，打开设置，并点击「已下载描述文件」：

![](https://image.peterjxl.com/blog/image-20250129150047-ut9hd0s.png)

‍

点击安装，并一步步确认（会要求输入锁屏密码，各种确认）：

![](https://image.peterjxl.com/blog/image-20250129150140-s37uln9.png)

‍

随后，我们打开设置，找到通用 → 关于本机：

![](https://image.peterjxl.com/blog/image-20250129150251-6b98dd2.png)

‍

拉到最下方，找到「证书信任设置」并点击：

![](https://image.peterjxl.com/blog/image-20250129150427-l4fo9yk.png)

‍

开启对证书的信任：

![](https://image.peterjxl.com/blog/image-20250129150516-ud4hqyf.png)

‍

至此，证书安装完成。

‍

### 使用梯子（可选）

需要说明的是，很多规则库都是需要翻墙后才能正常使用的。

我猜是因为，很多规则库都是放在 GitHub 上的，而 GitHub 是经常需要翻墙才能访问。

所以我们得在 Quantumult X 里添加机场的信息。

在主页面点击中间的加号（也可点击页面底部的加号）：

![](https://image.peterjxl.com/blog/image-20250129172418-2iaze2y.png)

‍

输入节点相关信息（右上角可能扫码添加）：

![](https://image.peterjxl.com/blog/image-20250129211718-deianl9.png)

‍

但使用梯子会带来一个问题：翻墙后，访问国内的网络，速度会慢不少。

目前，这个问题我还在研究中，暂无比较好的解决办法。

我现在用的是 [莫离 QuantumultX 配置](https://raw.githubusercontent.com/Moli-X/Resources/main/Rewrite/QuantumultX.conf)，导入后，不用梯子也能正常屏蔽知乎、B 站广告，对我来说够用了，就没加梯子。

‍

### 开启拦截

返回主页面，点击右上角的开关，即可开始接管流量：

![](https://image.peterjxl.com/blog/image-20250605163205-jio762d.png)

‍

至此，Quantumult X 就正式开始接管流量，并根据规则来拦截广告了。

你可以自行测试下，看看还有没有广告。

## 日常使用

在日常中，我一般都是全天 24 小时开启工具。无需保持后台常驻，只要你不主动去关开关，或者启用其它同类 APP，就会一直在，哪怕手机重启也在。

开启代理后，有些已经通过 WIFI 预加载的广告还是会显示，但一般过几次就没了，不会再显示（我在哔哩哔哩上遇到过）。

---

那耗电多吗？理论上肯定会比平时多一点，但也多不了多少。你担心的话，可以用一段时间看看耗电占比。

一般来说，代理软件的耗电量不大，就算设置里显示它们的耗电占比很高，也是因为开启这些软件之后，所有网络请求方面的电量消耗都被算到它们头上了，对于实际使用基本是没有影响。

---

关于通知：如果你切换了网络（例如从 WIFI 换到蜂窝网络），那么 Quantumult X 也会发送一个通知，建议还是关闭通知，不然太烦了。

![](https://image.peterjxl.com/blog/image-20250129213940-lcnr67w.png)

---

注意，如果你更换了配置（例如更换了规则），需要重新导入机场订阅，安装证书以及打开重写、MITM 开关。

‍

## 关于其他代理软件

除了 Quantumult X ，其他代理软件也能实现同样的效果，但貌似 Quantumult X 的生态更好，相关的规则更多。

如果你想用其他代理软件，也是可以的，例如 Stash，Clash，Shadowrocket，AdGuard 等，但请自行搜索相关的配置教程。

例如：[Shadowrocket-ADBlock-Rules-Forever](https://github.com/Johnshall/Shadowrocket-ADBlock-Rules-Forever)，14k Star，提供多种规则，拥有强劲的广告过滤功能。每日 8 时重新构建规则。
