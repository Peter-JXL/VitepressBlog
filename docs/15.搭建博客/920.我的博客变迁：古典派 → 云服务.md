# 我的博客变迁：古典派 → 云服务

最近将博客迁移到了云服务上，每年省下数千元，感慨万千。

本文就简单介绍下我的博客变迁过程，也分享下相关知识。



## 古典派

![](https://image.peterjxl.com/blog/image-20260215114203-9977r8r.png)

初次接触编程是在大学专业课上，学到了不少计算机基础知识。

毕业后，萌生出做一个博客的想法，然后花了几个月时间去折腾，包括：

- 买服务器，配置服务器，例如安装 Nginx（还是用的编译安装），搭建邮箱服务，安全加固等
- 选择博客框架，试了五六个，熟悉框架也花了很多时间
- 给博客加功能（CDN、评论区、搜索功能、数据统计，RSS 等），也花了很久去折腾

‍

在折腾的过程中，学到了很多知识。但花费也不少：

- 服务器：一开始是云服务器厂商搞活动，白嫖了一两年。但到期后就得续费了，而且也不便宜，一年就要 2.4k。虽然买 5 年的话有优惠，但也要 6.3k，年均一千多
- CDN：也就是加速服务，能让网站访问起来更快。它是按流量收费的，1T 要 126，去年花了上千元（还好平时有记账，不看不知道，一看吓一跳）
- 证书：一开始我不清楚有免费证书，买了正式证书，一年 300 多，而且我后面还用到了子域名，又要两个证书。

‍

以前我都不知道什么云存储，免费证书，公益 CDN 等，所以用的是我知道的做法。

但技术发展日新月异，各种基础设施开始丰富起来，之前的做法就有种古典的感觉，而且也很费钱。

![](https://image.peterjxl.com/blog/image-20260215114237-i4l2ec2.png)

随着和博友们的交流，了解了不少产品，然后开始将网站逐步迁移到免费服务上，下面一个一个来介绍。

‍

## 云服务


### 服务器​​

服务器支出是大头。

首先我为什么要用服务器？其实就是为了将网站文件放上去，然后供其他人访问。

这里服务器就是充当一个网盘的作用，所以我后面改为用对象存储服务（Cloud Object Storage，简称 COS）。

你可以简单理解为，COS 就是一个网盘，可以用来放网站文件，可以放图片（当图床）、音乐、电影、安装包等。

更关键的是：COS 很便宜，一年也就花个 8 毛钱。

将网站文件上传后，配置 CDN 访问它即可。

![](https://image.peterjxl.com/blog/image-20260213164217-uhkxjcw.png)

‍

### CDN

![](https://image.peterjxl.com/blog/image-20260213212202-f14rvvx.png)

之前有大厂搞活动，推出了 EdgeOne，号称永久免费，特地去抢了下注册码，然后开始将博客迁移过去。

迁移过去后，发现速度也还不错，而且自带安全防护，直接用就行，不用像之前那样做一大堆安全配置，还要担心被刷流量导致金钱上的损失。

这里没有打广告的意思，除了 EdgeOne，市面上还有 CloudFlare、Vercel 等厂家也提供类似的服务，只是它们在国外，访问起来慢一点点。

‍

### 证书

![](https://image.peterjxl.com/blog/image-20260215113252-tojxlvc.png)

以前比较笨，买了正式证书，其实个人博客用不着，用免费的测试证书就行了。

虽然有效期只有三个月，但是能配置自动更新。

这里不得不提一下，正式证书被放在最显眼的位置上，而免费证书则不显眼，还改名为了测试证书。我还以为仅仅是测试时能用，其他场合不能用呢 😅，不知道是不是厂商的小巧思。

‍

### Serverless

![](https://image.peterjxl.com/blog/image-20260215114503-adp7htw.png)

我之前的评论区功能、搜索功能，是搭建在自己的服务器上的。

但即使不用服务器，也可以用一些云函数部署。

简单来说就是直接提供代码给厂商，厂商就会帮你在云基础设施上弹性、安全地运行，我们不用关心服务器。

更多关于 Serverless 的介绍，推荐阅读：

- [小林 coding | 我又搭了个网站，1 分钟搞定！](https://mp.weixin.qq.com/s/nODrJob70rcON1QOkGRE9Q)
- [降本增效利器之 Serverless - iMaeGoo's Blog](https://www.imaegoo.com/2021/serverless/)

‍

## 最后

在知道众多云服务后，立马将花钱的项目都砍掉了，还好博客运行的时间也不久，没有花很多钱，之前订阅的服务器也可以退订，挽回了一点损失。

更重要的是，这让我以后不用再关心服务器的任何事情，只需关注写作本身即可，极大地节省了我的精力。

这让我回想起，知名博主阮一峰[曾写道](https://www.ruanyifeng.com/blog/2012/08/blogging_with_jekyll.html)：

> 喜欢写 Blog 的人，会经历三个阶段。
>
> 第一阶段，刚接触 Blog，觉得很新鲜，试着选择一个免费空间来写。
>
> 第二阶段，发现免费空间限制太多，就自己购买域名和空间，搭建独立博客。
>
> 第三阶段，觉得独立博客的管理太麻烦，最好在保留控制权的前提下，让别人来管，自己只负责写文章。

‍

如今回顾下我的写作历程，也确实是这样：

- 一开始先是在博客园、CSDN 等平台写
- 后来写的东西经常被删、限制太多，于是自己买服务器、域名，搭建独立博客
- 如今，全部托管到云服务上，我只负责写文章

不得感慨，要是我早点知道这些就好了。
